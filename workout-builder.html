What this page does

Build a list of exercises, and this page generates a URL that opens the Workout Timer with those exercises pre-loaded. No account needed, no server — everything is encoded in the URL itself.

URL format (V2 compact string)

The timer reads exercise data from the # fragment of the URL. The data is a semicolon-delimited string:

@<ref>[,<athlete>];<exercise1>;<exercise2>;...

Each exercise segment is comma-delimited:

<name>,<weight>,<sets>,<reps>,<work_secs>,<rest_secs>[,<flags>]

Fields

FieldTypeExampleNotes
refstringa1b2c36-char hex ID — any value works
athletestringJustinOptional, after ref with comma
namestringKettlebell SwingNo commas in name
weightstring20kg, BWNumber + unit, or BW
setsint4Number of sets
repsint10Reps per set
workint30Seconds of work per set
restint60Seconds of rest between sets
flagsstrings, o, sos = superset, o = optional

Supersets

To link exercises into a superset, add the s flag to every exercise in the group except the last one. The timer alternates sets across the linked exercises.

Bicep Curl,12kg,3,10,25,60,s;Tricep Ext,12kg,3,10,25,60

URL encoding

The timer accepts two encodings — use whichever is easier:

Plaintext — percent-encode the V2 string with encodeURIComponent() and append after #. Longer but human-readable and easy for any tool to produce.

Base64url — base64-encode the V2 string, then replace + with -, / with _, and strip trailing =. Shorter by ~20-25 chars.

iOS Calendar makes URLs clickable only up to ~350 characters. The char counts below each URL show whether you're within that limit.

Full example

A 2-exercise workout for "Justin":

@a1b2c3,Justin;Kettlebell Swing,20kg,4,10,30,60;Push-Ups,BW,3,15,30,60

As a plaintext URL:

workout-timer.html#%40a1b2c3%2CJustin%3BKettlebell%20Swing%2C20kg%2C4%2C10%2C30%2C60%3BPush-Ups%2CBW%2C3%2C15%2C30%2C60

For LLMs and tools

To programmatically generate a timer URL: build the V2 string per the spec above, percent-encode it, and prepend the timer base URL.
